## ADDED Requirements

### Requirement: Shell runtime configuration SHALL avoid hidden side effects

Managed shell helpers SHALL avoid installing packages, starting services, or fetching runtime dependencies as implicit side effects of entering a shell or project directory.

#### Scenario: A project directory enables a direnv layout

- **WHEN** a direnv layout is loaded
- **THEN** it SHALL configure environment variables and PATH entries only
- **AND** it SHALL NOT install missing dependencies automatically
- **AND** it SHALL NOT start container services automatically

#### Scenario: AI tooling exposes an adapter command

- **WHEN** AI tooling requires an adapter CLI
- **THEN** the adapter SHALL be installed through Nix package derivations when practical
- **AND** it SHALL NOT be launched through dynamic `npx -y` wrappers

### Requirement: Managed shell snippets SHALL rely on declared Home Manager environment

Shell snippets generated by Home Manager SHALL use Home Manager-declared runtime variables directly when those variables are part of the managed environment.

#### Scenario: zsh sources runtime secrets

- **WHEN** zsh starts under the managed Home Manager environment
- **THEN** it SHALL source secrets from `$XDG_CONFIG_HOME/.secrets.env` if that file exists
- **AND** it SHALL NOT use alternate fallback secret locations

### Requirement: Unused runtime services SHALL be removable from the module graph

Optional runtime services that are not working as intended SHALL be removable from default imports and enablement without affecting unrelated shell, editor, or CLI tooling.

#### Scenario: CLIProxy is unused

- **WHEN** Home Manager configuration is evaluated
- **THEN** CLIProxy SHALL NOT be imported, enabled, or installed by default
- **AND** no CLIProxy launchd/systemd service SHALL be generated
